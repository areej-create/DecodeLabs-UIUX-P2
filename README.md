## 📌 Project Overview

A low-fidelity wireframe set for an **E-commerce Homepage**, designed to solve a checkout abandonment problem caused by cluttered layout and confused navigation. The goal of this milestone was to strip away all visual decoration — color, imagery, branding — and prove the structure on its own: clear information hierarchy, logical user flow, and unmistakable Calls-to-Action (CTAs), built purely through layout and grayscale value.

**Tool:** Figma
**Fidelity:** Low-Fidelity (Skeleton stage — no color, no imagery)
**Palette Constraint:** Black, White, and Grey only
**Focus:** Information Architecture, User Flow, CTA Hierarchy

## 🧩 The Brief

**Scenario:** An e-commerce brand is losing customers during checkout. The layout is cluttered and users are confused — they need a clean, logical structure for their landing page before any visual styling happens.

**Mission:**
- Design the "skeleton" of the homepage, focused only on user flow and hierarchy
- Create low-fidelity wireframes for the Homepage
- Ensure clear, unambiguous CTA placement
- No color, no high-res imagery — grayscale placeholders only

---

## 🗺 Information Architecture Approach

| IA System | Where It's Used |
|---|---|
| **Hierarchical** | Header → Hero → Categories → Product Grid → Footer, ordered by visual importance |
| **Sequential** | Implied next-step flow from homepage → product → cart (Step 1 → 2 → 3 logic) |
| **Matrix** | Category chips and filters allow user-driven sorting/browsing |

**The 4 IA Systems applied:** Organization (categorizing products) · Labeling (clear section headers) · Navigation (predictable nav bar) · Searching (prominent search bar)

---

## 📐 Layout & Scanning Pattern

- **Pattern used:** F-Pattern — appropriate for a content-heavy homepage with multiple product blocks and descriptive sections, rather than a single hero/CTA Z-pattern layout
- **Primary scan zone:** Hero headline + single primary CTA, placed above the fold
- **Secondary scan zone:** Category shortcuts, using visual blocks rather than plain text links
- **Content blocks:** Product grid with consistent placeholder sizing to reduce cognitive load

---

## 📱 Wireframe Structure

### 1. Header / Navigation
- Logo placeholder (top-left)
- Full-width, prominent search bar
- Cart icon with badge indicator
- Minimum 44px tap targets on all interactive elements

### 2. Hero Section
- Benefit-driven headline placeholder + single primary CTA button
- No competing CTAs — one clear action above the fold
- Large image placeholder (no real imagery per brief constraints)

### 3. Category Shortcuts
- Visual category blocks (not text-only links) for faster recognition
- Avoids burying best-sellers behind a generic "Shop All" link

### 4. Product Grid
- Grayscale image placeholders, consistent card sizing
- Product name + price placeholder per card
- Framed with active whitespace to reduce visual clutter and decision fatigue

### 5. Footer
- Secondary navigation placeholder
- Contact / informational links grouped logically

## ✅ CTA & Friction Checklist

- [x] Single, unmistakable primary CTA above the fold
- [x] No unnecessary form fields or steps before product discovery
- [x] Visual (not text-only) category navigation
- [x] Consistent grid spacing to lower cognitive load
- [x] Grayscale-only palette to keep focus on structure, not decoration

---

## 🧱 Design Principles Applied

**Respect the Brain** — Every added element was tested against cognitive load; if it made the user think twice, it was simplified or removed.

**Blueprint First** — Structure was finalized before any aesthetic decision was considered, per the "Skeleton before Decoration" principle.

**Guide the Eye** — F-pattern scanning logic and active whitespace were used to direct attention toward the primary CTA and product grid.

**Audit Ruthlessly** — Every section was reviewed against the question: does this reduce or add friction to the path from landing to cart?
